# GNOME Manual Test Checklist

本清单用于在真实 `GNOME` 图形环境下做发布前人工验收。

## 环境准备

1. 已安装应用或可通过 `npm exec tauri dev` 启动
2. GNOME 会话可正常显示系统托盘区域或对应扩展
3. Home Assistant 地址、实体 ID、Token 已准备好
4. Linux Secret Service 可用

## 首次启动

1. 启动应用
2. 确认主窗口正常显示
3. 确认界面文案、布局、按钮都可见
4. 确认无明显白屏、空白区或样式错乱

## 配置页

1. 打开 `配置`
2. 输入 `baseUrl`
3. 输入 `climateEntityId`
4. 输入 Token 并保存
5. 重新打开配置页，确认留空 token 时不会覆盖已保存 token
6. 尝试清除 token，确认成功提示正常

## 主控制页

1. 点击 `刷新`
2. 确认连接状态刷新成功
3. 点击 `开机`
4. 点击 `关机`
5. 点击 `温度 +`
6. 点击 `温度 -`
7. 点击 `执行自动开机`
8. 确认消息提示与状态显示一致

## 托盘行为

1. 点击窗口关闭按钮，确认窗口隐藏而不是退出
2. 从托盘菜单点击 `打开`，确认主窗口恢复
3. 从托盘菜单点击 `配置`，确认窗口恢复并切到配置页
4. 最小化窗口，确认窗口隐藏到托盘
5. 从托盘菜单点击 `退出`，确认应用真正退出

## 导入导出

1. 导入旧版 `appsettings.json`
2. 确认普通配置字段被迁移
3. 若旧配置含明文 token，确认 token 被迁入密钥链
4. 导出新配置，确认导出文件中不包含 token

## 自启动

1. 在配置页开启 `系统登录后自动启动`
2. 退出应用并重新登录桌面会话
3. 确认应用自动启动
4. 再关闭该选项并重新登录验证关闭生效

## 风险观察

重点观察：

1. 托盘图标是否稳定显示
2. GNOME 下是否需要额外扩展才能稳定使用托盘
3. Secret Service 不可用时的错误提示是否清晰
4. 自启动是否受桌面会话或发行版差异影响
