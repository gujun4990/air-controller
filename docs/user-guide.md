# User Guide

## 应用用途

`AirController` 用于在桌面上快速控制 `Home Assistant` 中的空调实体。

你可以使用它：

1. 刷新空调状态
2. 开机
3. 关机
4. 调整目标温度
5. 配置自动启动与自动开机

## 安装方式

优先建议直接使用发布页中的安装包，而不是从源码启动。

### Linux

常见安装包格式：`.deb`、`.AppImage`

如果是 `.deb`：

```bash
sudo dpkg -i air-controller_<version>_amd64.deb
sudo apt-get install -f
```

### Windows

常见安装包格式：`.msi`、`.exe`

1. 双击安装包
2. 按安装向导完成安装
3. 从开始菜单启动 `AirController`

### macOS

常见安装包格式：`.dmg`

1. 打开 `.dmg`
2. 将 `AirController.app` 拖入 `Applications`
3. 从应用程序目录启动

如果首次打开被系统拦截，请到系统安全设置里允许打开。

## 首次使用

首次启动后，请先进入 `配置` 页面。

需要填写：

1. `HA 地址`
2. `空调实体 ID`
3. `Token`

建议再按需设置：

1. 是否系统登录后自动启动
2. 是否程序启动后自动开空调
3. 默认温度
4. 最小温度
5. 最大温度
6. 调温步长

保存后，Token 会进入系统密钥链，不会写入普通配置文件。

保存配置或保存 Token 后，应用会自动尝试验证与 Home Assistant 的连接，并在界面顶部显示结果。

## 主界面说明

主界面会显示：

1. 连接状态
2. 实体 ID
3. 电源状态
4. 当前温度
5. 目标温度
6. 消息提示

可执行操作：

1. `开机`
2. `关机`
3. `温度 +`
4. `温度 -`
5. `刷新`
6. `执行自动开机`

## 配置文件与 Token

普通配置保存在系统配置目录中的 `config.json`。

Token 不会写入这个文件，而是保存在系统密钥链中。

这意味着：

1. 导出配置时默认不包含 token
2. 迁移到新设备时通常需要重新填写 token

## 导入旧配置

如果你之前使用旧版 WPF 程序，可以在配置页导入旧版 `appsettings.json`。

迁移规则：

1. 普通配置字段会被迁移
2. 明文 token 会迁入系统密钥链
3. 加密 token 不会自动解密迁移，需要重新填写

## 托盘与退出行为

应用默认支持托盘常驻。

行为如下：

1. 点击窗口关闭按钮时，窗口会隐藏到托盘
2. 最小化时，也会隐藏到托盘
3. 托盘菜单可执行 `打开`、`配置`、`退出`

## 常见问题

### 1. 为什么保存后看不到 Token？

因为 Token 不写入普通配置文件，而是保存在系统密钥链。

### 2. 为什么 Linux 下无法保存 Token？

通常是因为系统密钥服务不可用，需要确认 `Secret Service` 可用。

### 3. 为什么关闭窗口后程序没有退出？

这是默认行为，程序会隐藏到托盘。需要通过托盘菜单选择 `退出` 才会真正退出。
